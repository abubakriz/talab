## Why Talab?

<CardGrid>
	<Card title="No try/catch blocks" icon="error">
		Network failures and HTTP status **codes** are returned as values, not exceptions.
	</Card>
	<Card title="Linear control flow" icon="right-arrow">
		Reduces nesting and keeps your business logic flat and readable.
	</Card>
	<Card title="Type-safe" icon="seti:typescript">
		Strong TypeScript inference directly on the response object.
	</Card>
	<Card title="Zero dependencies" icon="puzzle">
		A lightweight wrapper around native `fetch` with a tiny footprint (~1kb gzipped).
	</Card>
</CardGrid>
